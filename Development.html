Some Background on this Project


Web development has progressed greatly since the early days of basic HTML pages in the 90's and early 2000's. One of the more recent paradigms that has emerged in web development is the use of components to create interactive webpages with reusable and dynamic parts. In this particular project we will be implementing a component based webpage with React!

I built out an interactive website that gives you information about the various Star Wars films that have been released throughout the years. It also allows you to add the films you have seen to a list and it aggregates the total amount of time you have spent watching Star Wars films.



Architecture of the Webpage


Components


In this webapp, the only main component used was a "MovieItem" component. The purpose of this component was to provide and easy and repeatable way to generate each of the movie elements based off the movie list JSON for the primary part of the interface.

I also used the React Bootstrap component "Dropdown" to represent the filter and sort by values.


Props


Data from App.js is passed to the MovieItem component via props.


State


The user triggers state changes via interaction with the various buttons across the interface. Each button is tied to a different state change, for example there are 3 state variables representing each of the filter states, one for the sort by state and various buttons controlling the add to and remove from movie list functions. Each of these buttons is tied to its respective function, and triggers the state change when pressed.


Filtering


The user has multiple options to filter from. This was implemented using custom filtering functions implemented in the code.


Sorting


The user has multiple options to use to sort the components. This was implemented using custom filtering functions implemented in the code.



Closing Remarks


While this is a simple demonstration of the capabilities of React, the implications of being able to use components and states within building a WebApp are profound.
